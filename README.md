## <img src="./pics/icon/motivation.png" width="25" /> 项目动机

> **Rebuttal 不是简单地"反驳审稿人"**，而是在极短篇幅内重新组织证据、澄清误解、补足实验，并让 AC 和审稿人看到论文仍然值得被接收。

本项目整理了 AI 顶会投稿中常见的 rebuttal 场景，将每一类问题总结为四部分：

<div align="center">

| 组成部分 | 说明 |
| --- | --- |
| **审稿人问题** | 典型的审稿意见或质疑 |
| **反面回答** | 容易踩坑的回复方式 |
| **推荐回答** | 更有效的回应策略 |
| **核心点** | 关键要点总结 |

</div>

<details>
<summary><b>⚠️ 免责声明</b></summary>

<br/>

本项目列举的所有技巧**仅供参考**，并不保证正确，不保证适用于所有会议、所有领域或所有审稿场景。不同会议的 rebuttal 规则、字数限制、是否允许新增实验和是否允许外链可能不同，请以会议官方说明为准。

所有内容仅仅来自于笔者的个人经验、互联网数据、笔者团队日常科研工作中的相关积累，以及笔者团队身边各位大佬的言传身教。有任何问题，欢迎提交 Issue 或 PR。另本项目所用徽章来自互联网，如侵犯了您的图片版权请联系我们删除，谢谢。

</details>
